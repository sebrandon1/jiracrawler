# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in jiracrawler, please report it responsibly.

### How to Report

- **GitHub Security Advisories**: Use [GitHub Security Advisories](https://github.com/sebrandon1/jiracrawler/security/advisories/new) to report vulnerabilities privately.
- **Email**: Alternatively, contact the maintainers directly via their GitHub profile.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix and disclosure**: Coordinated with reporter

## Scope

This policy applies to the jiracrawler CLI tool and its dependencies.

## Supported Versions

Only the latest release is actively supported with security updates.
